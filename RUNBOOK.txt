BLACKBOXNLP-2026 — SHUFFLED-PROTOCOL PIPELINE (v2, 2026-07-11)
Run on the pod, top to bottom. Supersedes the first-slice runbook after the ordering
audit ESCALATED Marathi (C2ST AUC 0.942) and marginally Bengali (0.553); Hindi clean
(0.498). Decision: ALL extractions use --shuffle-stories (random sample of the full
corpus). Old first-slice numbers are kept ONLY as the ordering-robustness baseline.

TWO PROTOCOL RULES (breaking either invalidates the run):

  RULE 1 — occurrence protocol is min-occ = 60, fixed-occ = 60 EVERYWHERE.
    (The previous runbook said 100 — that was drift. The validated 2026-07-05 run and
    the paper both use 60; verified from the stored signatures: n_occ_used == 60 for
    every concept in every concept_sig_*.npz. Using 100 would change the concept sets
    (130/198) and silently break the old-vs-new ordering comparison.)

  RULE 2 — SEED PLAN for --shuffle-seed:
        language   trained model   floor model(s)              shuffle-seed
        Hindi      hindi_54M       random_hindi, random_hindi_b     101
        Marathi    marathi_54M     random_marathi                   202
        Bengali    bengali_54M     random_bengali, random_bengali_b 303
    SAME seed within a language: trained and floor must see the byte-identical story
    sample (the ladder is a paired design; "never mix protocols across trained/floor").
    DIFFERENT seeds across languages: Marathi and Bengali both have exactly 2,000,000
    stories, so a shared seed would produce the IDENTICAL permutation and preserve
    cross-corpus index alignment — the artifact we are breaking.

ORDERING INSIDE EACH LANGUAGE BLOCK: everything that needs the 25 GB activation
memmap runs BEFORE that model's rm -rf — probes (SAE + raw), T5 honest ceiling,
A8 hierarchy eval, and the 2 GB eval slice. Deleting early = full re-extraction.

Folders on the pod (unchanged):
  code         /workspace/projects/BLACKBOXNLP-2026
  checkpoints  /workspace/checkpoints/checkpoints-pytorch
  activations  /workspace/activations      (25 GB memmaps, deleted per model)
  cache        /workspace/cache            (HF downloads; each corpus ~5-6 GB)
  outputs      /workspace/outputs          (SAEs, signatures, RSA json, hier json)

RAM note: each corpus is ONE train.json (5-6 GB); the reader parses it wholesale and
--shuffle-stories holds the story list in RAM. Budget ~40-60 GB RAM headroom. Disk
peak per block is ONE memmap (~26 GB) + eval slices (~2 GB each) + cache.

import torch, itertools, config as C
vs = {}
for m in ["random_hindi","random_marathi","random_bengali","random_hindi_b","random_bengali_b"]:
    sd = torch.load(C.CHECKPOINTS[m], map_location="cpu", weights_only=False)["model"]
    v = torch.cat([p.flatten() for p in sd.values()]).float()
    vs[m] = v / v.norm()
for a, b in itertools.combinations(vs, 2):
    print(f"V1 {a:16s} vs {b:16s}: cos = {float(vs[a] @ vs[b]):+.6f}")
PY
